# 🌍 Change 1% — المنصة العالمية لإنقاذ الكوكب

منصة ويب مفتوحة المصدر تعرض كوكب الأرض بشكل تفاعلي ثلاثي الأبعاد، وتتيح لأي شخص في العالم توثيق قضية إنسانية أو بيئية أو صحية أو تعليمية على موقعها الجغرافي الحقيقي، ليراها ويتفاعل معها المجتمع مباشرة (دعم، تعليقات، تحديث حالة).

الهدف: تحويل "قضية محلية صغيرة" إلى "قضية مرئية عالمياً" — بحيث لو كل شخص غيّر 1% بس من العالم من حوله، النتيجة تتراكم لتغيير حقيقي.

## 🧰 التقنيات المستخدمة

- **HTML + CSS + JavaScript (Vanilla)** — بدون أي إطار عمل (Framework)، ملف واحد فقط يعمل مباشرة في المتصفح.
- **[CesiumJS](https://cesium.com/platform/cesiumjs/)** — لعرض الكرة الأرضية ثلاثية الأبعاد والتعامل مع الإحداثيات الجغرافية.
- **[Firebase Realtime Database](https://firebase.google.com/docs/database)** — لتخزين ومزامنة البيانات لحظياً بين كل المستخدمين.
- **[Nominatim API](https://nominatim.org/)** (OpenStreetMap) — تحويل إحداثيات (Lat/Lng) إلى اسم دولة/مدينة، مجاني بالكامل.
- **[ImgBB API](https://api.imgbb.com/)** — استضافة صور مجانية لبلاغات القضايا.
- **[Font Awesome 6](https://fontawesome.com/)** و **[Google Fonts - Cairo](https://fonts.google.com/specimen/Cairo)** — الأيقونات والخط العربي.

لا حاجة لأي عملية بناء (Build Process) أو npm — الملف شغال فوراً بمجرد فتحه في أي متصفح أو رفعه على أي استضافة ثابتة (Static Hosting).

## 🚀 خطوات التشغيل خطوة بخطوة


افتح `change1percent.html` وابحث عن السطر:

```javascript
const firebaseConfig = {
    apiKey: "YOUR_API_KEY",
    authDomain: "YOUR_PROJECT.firebaseapp.com",
    databaseURL: "https://YOUR_PROJECT-default-rtdb.firebaseio.com",

    messagingSenderId: "YOUR_SENDER_ID",
    appId: "YOUR_APP_ID"

```

واستبدل كل قيمة بالقيم الحقيقية من مشروع Firebase بتاعك.

### الخطوة 4: (اختياري) إعداد رفع الصور

ابحث عن:

```javascript

```

وضع مفتاح ImgBB بتاعك (خطوات الحصول عليه بالأسفل). لو سبتها زي ما هي، الموقع هيشتغل عادي بس بدون خاصية رفع الصور.

```javascript

```

غيّرها لكلمة مرور خاصة بيك قبل أي استخدام حقيقي.

### الخطوة 6: التشغيل محلياً

أبسط طريقة، افتح الملف مباشرة في المتصفح:

```bash

open change1percent.html

# على Windows
start change1percent.html

# أو شغّل سيرفر محلي بسيط (مفضّل، لتجنب مشاكل CORS في بعض المتصفحات)

# ثم افتح http://localhost:8000/change1percent.html
```

أي استضافة ثابتة تكفي، على سبيل المثال:

**GitHub Pages:**
```bash
git add change1percent.html
git commit -m "نشر المنصة"

```

```bash

```

## 🔥 إعداد Firebase بالتفصيل

### 1. إنشاء المشروع

1. ادخل على [console.firebase.google.com](https://console.firebase.google.com)
2. اضغط **"إنشاء مشروع" (Add project)**
3. اختر اسم للمشروع (مثلاً `change1percent`)
4. تقدر تعطّل Google Analytics لو مش محتاجه (اختياري)
5. اضغط **إنشاء المشروع**

### 2. تفعيل Realtime Database

1. من القائمة الجانبية، اختر **Build → Realtime Database**
2. اضغط **"إنشاء قاعدة بيانات" (Create Database)**
3. اختر موقع السيرفر الأقرب جغرافياً لجمهورك المستهدف
4. ابدأ في **وضع الاختبار (Test Mode)** مبدئياً للتطوير فقط (**ليس للإنتاج** — راجع قسم الأمان بالأسفل)

### 3. الحصول على بيانات الإعداد (Config)

1. من الصفحة الرئيسية للمشروع، اضغط على أيقونة **⚙️ (Project Settings)**
2. انزل لقسم **"Your apps"** واضغط أيقونة **الويب `</>`**
3. سجّل اسم للتطبيق واضغط **Register app**
4. هيظهرلك كائن `firebaseConfig` — انسخه كامل والصقه في الملف مكان القيم الافتراضية

### 4. ضبط قواعد الأمان (Security Rules) — **مهم جداً**

من تبويب **Rules** داخل Realtime Database، استخدم كحد أدنى:

```json
{
  "rules": {
    "global_issues": {
      ".read": true,
      ".write": true,
      "$issue_id": {
        "upvotes": {
          ".validate": "newData.isNumber() && newData.val() >= 0"
        },
        "comments": {
          "$comment_id": {
            "text": {
              ".validate": "newData.isString() && newData.val().length <= 500"
            }
          }
        }
      }
    }
  }
}
```

هذه القواعد لسه أساسية وليست مثالية للإنتاج الكامل — راجع قسم الأمان بالأسفل لتفاصيل أدق.

## 🖼️ إعداد رفع الصور (ImgBB)

1. ادخل على [api.imgbb.com](https://api.imgbb.com/)
2. سجّل حساب مجاني
3. من صفحة **"About"** أو **"API"** هتلاقي **API Key** خاص بيك
4. انسخه والصقه في الملف:

```javascript
const IMGBB_API_KEY = "ضع_المفتاح_هنا";
```

> ملاحظة: لو المفتاح متسيب على القيمة الافتراضية `YOUR_IMGBB_API_KEY`، الموقع هيتجاهل رفع الصور تلقائياً وهيكمل الحفظ عادي من غير صورة (مفيش Error هيظهر للمستخدم).

## 🛡️ نظام صلاحيات الأدمن

النظام الحالي بسيط عمداً عشان يكون سهل الفهم للمساهمين الجدد:

- زر **"دخول الإدارة"** أعلى القائمة الجانبية.
- عند الضغط، بيطلب كلمة مرور (المخزنة في المتغير `ADMIN_PASSWORD`).
- بعد الدخول الصحيح، بتظهر لوحة تحكم صغيرة جوه أي قضية تسمح بـ:
  - تغيير الحالة (جديدة / عاجلة / قيد المراجعة / تم حلها)
  - حذف القضية نهائياً

### ⚠️ حدود هذا النظام (يجب معرفتها)

هذه الطريقة **حماية بسيطة على مستوى الواجهة فقط (Client-Side)** ومناسبة لمشروع تجريبي أو مرحلة MVP، لكنها **غير كافية للإنتاج الحقيقي** للأسباب التالية:

- كلمة المرور موجودة كنص صريح داخل كود الجافاسكريبت، وأي شخص يقدر يفتح "أدوات المطوّر" (Developer Tools) في المتصفح ويقرأها مباشرة.
- التحقق بيحصل في المتصفح فقط، مش في السيرفر، فنظرياً أي شخص متمرّس يقدر يتجاوزه ويكتب مباشرة في قاعدة البيانات.

**للاستخدام الحقيقي، يُنصح بشدة بالتحويل إلى:**

1. **[Firebase Authentication](https://firebase.google.com/docs/auth)** (تسجيل دخول Email/Password أو Google) بدلاً من كلمة مرور ثابتة.
2. **Firebase Security Rules** تتحقق من هوية المستخدم على مستوى السيرفر، مثال:

```json
{
  "rules": {
    "global_issues": {
      "$issue_id": {
        "status": {
          ".write": "auth != null && root.child('admins').child(auth.uid).exists()"
        }
      }
    }
  }
}
```

3. جدول `admins` منفصل في قاعدة البيانات يحدد أي مستخدمين (بالـ UID) عندهم صلاحية أدمن فعلياً.

## 🗂️ شرح هيكل البيانات (Data Model)

كل قضية مخزنة تحت المسار `global_issues/{issue_id}` بالشكل التالي:

```json
{
  "global_issues": {
    "-Nxxxxxxxxxxxxxxxxx": {
      "title": "نقص مياه شرب نظيفة في القرية",
      "description": "شرح تفصيلي للمشكلة والاحتياجات...",
      "lat": 30.0444,
      "lng": 31.2357,
      "status": "عاجلة",
      "category": "إنسانية",
      "address": "📍 مصر - القاهرة | حي معين",
      "image": "https://i.ibb.co/xxxxx/photo.jpg",
      "upvotes": 12,
      "timestamp": 1731234567890,
      "comments": {
        "-Mxxxxxxxxxxxxxxxxx": {
          "text": "أقدر أساعد بتوفير خزانات مياه",
          "timestamp": 1731234999999
        }
      }
    }
  }
}
```

| الحقل | النوع | إلزامي؟ | الوصف |
|---|---|---|---|
| `title` | string | نعم | عنوان القضية |
| `description` | string | نعم | الشرح التفصيلي |
| `lat` / `lng` | number | نعم | الإحداثيات الجغرافية |
| `status` | string | نعم | إحدى القيم: `جديدة`, `عاجلة`, `قيد المراجعة`, `تم حلها` |
| `category` | string | نعم | إحدى القيم: `إنسانية`, `بيئية`, `صحية`, `تعليمية` |
| `address` | string | لا | عنوان بشري مُولّد تلقائياً |
| `image` | string / null | لا | رابط صورة من ImgBB |
| `upvotes` | number | نعم (افتراضي 0) | عدد الأشخاص الداعمين للقضية |
| `timestamp` | number | نعم | وقت الإنشاء (Unix ms) |
| `comments` | object | لا | تعليقات فرعية، كل تعليق فيه `text` و `timestamp` |

## 📱 التوافق مع الموبايل

- على الشاشات **أوسع من 860px**: القائمة الجانبية ثابتة على اليمين والخريطة تملأ باقي الشاشة.
- على الشاشات **أضيق من 860px**:
  - القائمة الجانبية تختفي تلقائياً وتتحول لقائمة سفلية (Bottom Sheet).
  - زر عائم دائري (☰) في الزاوية يفتح/يقفل القائمة.
  - شريط الإحصائيات العلوي يتصغّر ليتناسب مع عرض الشاشة.

> ملاحظة أداء: مكتبة CesiumJS ثقيلة نسبياً على أجهزة الموبايل الأقدم؛ لو واجهت بطء، فعّل خيار تقليل الجودة (`viewer.scene.globe.maximumScreenSpaceError`) أو استخدم Cesium Ion لتحسين الأداء.

## 💡 أفكار للمساهمة والتطوير المستقبلي

المشروع مفتوح المصدر ومرحّب بأي مساهمة. أفكار مقترحة لمن يريد المساهمة:

- [ ] **Firebase Authentication حقيقي** بدل كلمة مرور الأدمن الثابتة.
- [ ] **كشف تكرار البلاغات** — تنبيه لو في بلاغ قريب جغرافياً من بلاغ موجود بالفعل (لتفادي التكرار).
- [ ] **نظام تحقق (Verification)** — علامة "موثّق ✅" على القضايا اللي تم التأكد منها من جهة موثوقة.
- [ ] **دعم لغات متعددة** — إضافة تبديل عربي/إنجليزي (i18n) للوصول لجمهور أوسع.
- [ ] **إشعارات** — تنبيه المستخدمين اللي دعموا قضية معينة عند تغيير حالتها.
- [ ] **تصدير بيانات** — إمكانية تصدير قائمة القضايا كـ Excel/PDF للجمعيات الخيرية.
- [ ] **وضع عدم الاتصال (Offline)** — تخزين مؤقت محلي للقضايا القريبة.
- [ ] **تحسين أداء الموبايل** — تفعيل Level of Detail (LOD) في Cesium لأجهزة أضعف.

للمساهمة:
```bash
1. اعمل Fork للمستودع
2. اعمل branch جديد: git checkout -b feature/اسم-الميزة
3. اعمل commit للتعديلات: git commit -m "إضافة: وصف الميزة"
4. ادفع للـ branch: git push origin feature/اسم-الميزة
5. افتح Pull Request
```

## 🔧 حل المشاكل الشائعة (Troubleshooting)

| المشكلة | السبب المحتمل | الحل |
|---|---|---|
| الخريطة مش بتظهر / بيضاء | مشكلة في تحميل CesiumJS (اتصال إنترنت أو CDN محجوب) | تأكد من الاتصال بالإنترنت وإن روابط CDN مش محجوبة على شبكتك |
| البلاغات مش بتتحفظ | إعدادات `firebaseConfig` غير صحيحة أو Security Rules ترفض الكتابة | تأكد من نسخ الإعدادات بدقة ومن قواعد الأمان في Firebase Console |
| العنوان بيظهر "منطقة غير مأهولة" دايماً | نفس الإحداثيات مكاش locally أو Nominatim مش راجع بيانات للموقع ده | جرّب موقع تاني، أو تأكد إن الـ User-Agent مش محجوب من Nominatim |
| الصور مش بترفع | مفتاح ImgBB غير صحيح أو لسه على القيمة الافتراضية | تأكد من حط مفتاح ImgBB صحيح فعّال |
| زر المشاركة بيعمل نسخ بدل مشاركة مباشرة | المتصفح أو الجهاز لا يدعم `navigator.share` (شائع على أجهزة الديسكتوب) | هذا سلوك متوقع؛ الرابط بينسخ للحافظة تلقائياً بديلاً |
| لوحة الأدمن مش ظاهرة | لسه معملتش تسجيل دخول، أو الجلسة (Session) انتهت بإغلاق التاب | اضغط "دخول الإدارة" وأدخل كلمة المرور الصحيحة |
